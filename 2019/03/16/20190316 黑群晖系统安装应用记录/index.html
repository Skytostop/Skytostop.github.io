黑群晖系统安装应用记录

发表于 2019-03-16 | 更新于 2019-06-16 | 分类于 善用网络 | 评论数： | 阅读次数：
本文字数： 1.7k | 阅读时长 ≈ 3 分钟

这两天折腾搞了个黑群晖，自己是小白，所以在这一块都是照着网上的教程一步一步操作的，所以有必要自己再记录下大致的思路过程，方便下次重新安装。拿之前第一次搭建 Hexo 博客来说，虽然最后成功了，但是如果让我再操作一遍，基本还是要一步步对照指南，因为我对那些基本的各种命令根本看不懂，只能一步一步模仿，这样只要教程稍微跳过了一些作者以为是很简单的步骤，但是对于小白可能就惨了，所以我选择记录下。

因为生命不息，折腾不止呀！

要说下的是，我这篇主要是思路记录，是给自己参考的，我这次系统安装也主要是参考网友的教程来搭建的。

这次操作是在 Windows 环境下，需要下载的文件有

1.群晖系统，推荐 DSM 6.1.7-15284 Update3 img 和 pat 结尾文件
2.ChipEasy 芯片无忧，查找 U 盘的 VID 和 PID
3.DiskGenius，复制 grub.cfg 到桌面
4.Notepad++，打开编辑 grub.cfg
5.Win32 Disk Imager，将系统写入U盘

第 1 步 修改引导配置文件

1.1 用 ChipEasy 芯片无忧 查找U盘的 VID 和 PID，并做记录。
1.2 用 DiskGenius 编辑配置文件 grub.cfg，
菜单栏选择硬盘——打开虚拟硬盘文件——选择刚刚下载的img文件，在左侧已打开的 img 文件——找到 grub.cfg——右击选择复制到桌面，
1.3 修改 VID 和 PID
打开桌面的 grub.cfg 文件选择用 Notepad++ 打开，找到下面两行修改 0x 后面值为刚刚 ChipEasy 读取的值。

set vid=0x058f U 盘的 VID 实体机必须修改，虚拟机无需修改
set pid=0x6387 U 盘的 PID 实体机必须修改，虚拟机无需修改

1.4 替换 grub.cfg 原文件
把已经编辑好的 grub.cfg 文件拖动到 DiskGenius 刚刚的位置, 并且替换之前的文件。
Tips: 以上步骤操作结束后，记得关闭 DiskGenius，否则 U 盘写入时会出错。

第 2 步 制作引导盘

运行 Win32 Disk Imager 磁盘映像工具，选择自己 U 盘符，选择之前下载的 synoboot.img 文件，再点击写入，直至写入成功大概1分钟不到。

第 3 步 U 盘安装群晖环境

插入U盘至旧主机或者自己搭的 NAS，进入 Bios 先选择 U 盘启动，不清楚的同学这一步可以谷歌一下，默认选择第1个 DS3617xs 6.1 Baremetal with Jun’s Mod v1.02-alpha 回车「其实这个选择的界面大概只出现 0.5 秒就跳过去了，默认就好」。
出现下图则表示安装成功，大概 1 分钟就可以。这里我说一下，其实只要出现红线框起来的部分也有可能是已经安装了，我却一直傻等，后来网上查才知道已经成功了。

第 4 步 安装群晖系统

4.1 浏览器地址栏输入 find.synology.com 查找系统
4.2 安装 DSM——手动安装——选择 .pat 文件，安装会清空硬盘文件
4.3 设置服务器名称、用户名、密码
4.4 原装群晖用 QuickConnect 账户登陆，黑群晖选择跳过此步骤
至此，群晖系统已安装完成，需要提醒的是不要轻易升级，去控制面板里系统计划关闭 DSM 自动更新选项。

第 5 步 群晖 DSM 添加硬盘存储空间的办法

5.1 进入系统后，点击左上角方块菜单——选择存储空间管理员，按提示添加硬盘即可，这里同样会清空硬盘文件，因为群晖系统格式和电脑格式不一样，所以一定会清空一下，具体详细设置大家可以谷歌或参考下这篇文章——群辉Synology DSM 添加 HDD 硬盘/存储空间的方法

终于整理完啦，自己梳理一次感觉清晰多了，这样下次就可以按照整理的步骤快速进行了，当然如果不想折腾又有这方面需求的，建议直接去群晖官网了解详情，可以去天猫货京东旗舰店购买。
